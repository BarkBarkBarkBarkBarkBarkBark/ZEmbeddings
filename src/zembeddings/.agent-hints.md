# For AI assistants navigating this package.

## Pipeline execution order

1. `params.py` — load/merge parameters
2. `tokenizer.py` — text → causal sliding windows
3. `embeddings.py` — windows → embedding vectors (cached)
4. `metrics.py` — embeddings → kinematic metrics
5. `kalman.py` — embeddings + distances → Kalman filter results
6. `pipeline.py` — orchestrates steps 1–5, writes outputs
7. `io.py` — YAML + Markdown serialisation
8. `ingest.py` — corpus-specific parsing (separate from pipeline)
9. `database.py` — pgvector storage (optional)

## Dependency direction

```
params ← tokenizer ← embeddings
                          ↓
                       metrics
                          ↓
                       kalman
                          ↓
pipeline → [tokenizer, embeddings, metrics, kalman, io]
ingest → [database (optional)]
```

## Key invariants

- All embedding vectors are L2-normalised → dot product = cosine sim
- cosine_distance = 1 - dot(e_t, e_{t-1}), range [0, 2]
- First element of all derivative arrays is NaN (no predecessor)
- Boundary detection uses CAUSAL running mean/std, not global
- Kalman filter uses constant-velocity model (predict-then-correct)
- Window indices are 0-based, contiguous
- PARAMS dict keys match YAML config keys exactly

## Common pitfalls

- Don't import openai/psycopg2/sentence_transformers at module level
- Don't compute global statistics and apply retroactively (breaks causality)
- Don't mutate PARAMS — always deep-copy via get_params()
- Experiment YAML 'experiment' section is metadata, not a PARAMS key
