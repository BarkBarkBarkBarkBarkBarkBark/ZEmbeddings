## What this project does

ZEmbeddings treats conversation transcripts as **trajectories through
semantic embedding space** and computes kinematic metrics (velocity,
acceleration, jerk) plus Kalman-filter boundary detection. Think
"physics of meaning" — classical mechanics applied to language.

## Critical rules

1. **MANIFESTO.md is ground truth.** If code disagrees with the
   manifesto, the code has a bug.

2. **Everything is causal.** At time t, only steps 0..t are available.
   No future tokens, no global statistics applied retroactively, no
   bidirectional smoothing.

3. **PARAMS is the single source of truth for parameters.** Never
   mutate PARAMS directly — use `get_params()` for a deep copy.
   YAML configs deep-merge over defaults.

4. **No API calls in tests.** Use synthetic L2-normalised vectors.
   Mock external services.

## Module map (in pipeline order)

| Module | One-line summary |
|--------|-----------------|
| `params.py` | Central parameter dict + YAML loader |
| `tokenizer.py` | Causal sliding-window tokenisation (tiktoken) |
| `embeddings.py` | OpenAI API + local sentence-transformers, .npz cache |
| `metrics.py` | Velocity, acceleration, jerk, EMA, boundaries, fixation, returns |
| `kalman.py` | Scalar + vector Kalman filters, Mahalanobis violation detection |
| `pipeline.py` | Orchestrator: tokenize → embed → metrics → kalman → output |
| `io.py` | YAML + Markdown report writers |
| `ingest.py` | Corpus parser (no_media format → postured text files) |
| `database.py` | pgvector storage (optional, gated behind extras) |

## Parameter override flow

```
PARAMS (Python) → default.yaml → experiments/*.yaml → get_params(**overrides)
```

## Key dataclasses

- `TrajectoryMetrics` — 15+ metric arrays for one embedding trajectory
- `KalmanResult` — innovations, Mahalanobis distances, violation flags
- `PipelineResult` — all pipeline outputs in one inspectable object
- `TokenizedTranscript` — token IDs + list of `Window` objects
